## 🔍 Vulnerability Summary

### 1. Critical: API Key Exposure (Direct Access)
**Issue:** `VITE_GEMINI_API_KEY` and `VITE_GOOGLE_TRANSLATION_API_KEY` are used directly in frontend composables.
- **Impact:** Anyone can extract these keys from your website's source code and use your paid AI/Translation quota, potentially leading to massive costs or service exhaustion.
- **Locations:** 
  - `src/composables/useAISummary.ts`
  - `src/composables/useOcrPipeline.ts`

### 2. High: Google Maps Key Exposure
**Issue:** The Maps API key is available in the public bundle.
- **Impact:** While intended for the client, if not restricted, it can be used on other websites.
- **Location:** `plugins/googleMapsLoader.ts`

### 3. Medium: Supabase Row Level Security (RLS)
**Issue:** Use of `VITE_SUPABASE_ANON_KEY`.
- **Impact:** If RLS is not strictly configured on the backend, any user can potentially read or modify any table in your database.
- **Location:** `plugins/supabaseClient.ts`

### 4. Low: Frontend Obfuscation
**Issue:** `vite.config.ts` has `keep_fnames: true` and `keep_classnames: true`.
- **Impact:** Makes reverse-engineering the app's logic significantly easier for attackers.

---

## 🛠️ Recommended Action Plan

### Step 1: Move "Secret" Keys to the Backend
Move all AI and Translation logic to **Supabase Edge Functions**.
1.  **Create Function:** `supabase functions new generate-summary`
2.  **Move Logic:** Copy the Gemini logic from `useAISummary.ts` to the function.
3.  **Secure Key:** `supabase secrets set VITE_GEMINI_API_KEY=your_key`
4.  **Call from App:** Use `supabase.functions.invoke('generate-summary')` in your Vue code.

### Step 2: Restrict "Public" Keys
1.  **Google Cloud Console:** Add "API Restrictions" to your Maps key, limiting it to **Android Apps** (using your SHA-1 fingerprint) and **Websites** (using your domain).
2.  **Supabase Dashboard:** Verify that "Enable RLS" is checked for **all** tables and that you have no "Permissive" policies for sensitive data.

### Step 3: Harden Production Builds
Update `vite.config.ts`:
```typescript
build: {
  terserOptions: {
    mangle: {
      keep_fnames: false, // Set to false for production
      keep_classnames: false, // Set to false for production
    }
  }
}
```

### Step 4: Secure .env Management
- **NEVER** commit your `.env` to Git.
- Use **Environment Secrets** in Vercel or GitHub Actions to inject keys during the build process.
